## 4. Ошибки и неточности ИИ, которые были исправлены вручную

### 4.1. Ошибка при сохранении модели в JSON

При сохранении структуры WANN возникла ошибка:

```text
TypeError: Object of type int64 is not JSON serializable
```

Причина заключалась в том, что некоторые значения внутри структуры сети имели тип `numpy.int64`, который нельзя напрямую сохранить через `json.dump`.

После проверки была добавлена функция преобразования NumPy-типов в обычные Python-типы:

```python
def convert_to_json_serializable(obj):
    if isinstance(obj, np.integer):
        return int(obj)
    elif isinstance(obj, np.floating):
        return float(obj)
```

После исправления структура WANN успешно сохранилась и загрузилась.

### 4.2. Необходимость проверки загруженной модели

Первоначально было недостаточно просто сохранить файл модели. Нужно было дополнительно проверить, что после загрузки сеть действительно работает так же, как до сохранения.

Поэтому была добавлена проверка загруженного агента. Загруженная сеть показала:

```text
Loaded mean reward: 500.0
Loaded std reward: 0.0
Loaded hidden nodes: 3
Loaded connections: 14
```

Это подтвердило корректность сохранения и загрузки.

### 4.3. Подбор структуры экспериментов

Сначала было неочевидно, какие эксперименты лучше использовать для выполнения требований работы. После анализа требований были выбраны следующие эксперименты:

- основной WANN;
- два независимых запуска с разными seed;
- baseline Random Search;
- абляция без мутации активации;
- сравнение размера популяции;
- сравнение вероятностей мутации;
- проверка устойчивости к шуму наблюдений.

Такой набор экспериментов позволил выполнить требования по baseline, ablation, hyperparameter comparison, robustness и reproducibility.

### 4.4. Предупреждение Matplotlib

При построении boxplot для проверки устойчивости появилось предупреждение Matplotlib о параметре `labels`. Оно не влияло на результат эксперимента, поэтому график был оставлен без изменения. При необходимости параметр можно заменить на `tick_labels`.

### 4.5. Упрощение блок-схемы

Сначала блок-схема могла получиться слишком подробной. Для отчёта была выбрана более простая схема, где показаны только основные этапы программы: задание параметров, инициализация популяции, оценка fitness, эволюционный цикл, тестирование, сравнение, проверка устойчивости, сохранение модели и анализ результатов.
